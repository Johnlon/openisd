2.Cracked-open evidence → analysis → recommendations

What I extracted from the open source, the live-site JS and the alignment math — each with a link to the actual source, an analysis, and a concrete recommendation for OpenISD.

F1 — The f3 = 81 Hz "discrepancy" is a model choice, not an OpenISD bug BUG?→NO

The lautsprechershop closed-box tool's inline JS (verbatim) uses a leakage-corrected empirical model, not the ideal lossless one:

qr = (1/qts)/(1/qb - .1);        // qb = target Qtc; "-0.1" = leakage term (Gaedtke)
fb = qr*fs;  vr = qr*qr-1;  Vb = vas/vr;               // → Vb ≈ 10 L (not 12.2)
f3 = 0.824 * fs/qts * (1 + 2*(qb-0.65)*(qb-0.65));      // empirical → 81 Hz

Analysis: the -0.1 leakage term shrinks the box (Vb 10 vs 12.2) and f3 is a curve-fit, not the true −3 dB. Independently, Scimpy's sealed model uses the theoretical F3: F3 = fc·√((1/Qtc²−2+√((2−1/Qtc²)²+4))/2), which equals fc exactly at Qtc=0.707 — matching OpenISD's 68.8 Hz. So OpenISD is correct; lautsprechershop simply models leakage.

Recommendation: close the oracle-report bug flag — not an OpenISD bug. But OpenISD only shows fc; add a proper F3 (−3 dB) read-out using Scimpy's closed-form (F3 ≠ fc for non-Butterworth Qtc). Reimplement from the formula (physics, not GPL code).

F2 — Ready-made vented alignment formulas (BACKLOG P1) GAPOPPORTUNITY

QSpeakers' optimizer.cpp contains closed-form Vb/Fb for the classic alignments OpenISD's backlog wants ("expand vented alignment presets"):

Bessel:      Vb = 8.0707·Qts^2.5848·Vas       Fb = 0.3552·Qts^-0.9649·Fs
Keele-Hoge:  Vb = 5.2358·Qts^2.1687·Vas       Fb = Fs
Bullock:     Vb = Qts·Vas·(4.96·Qts-0.136)    Fb = Fs
Legendre:    Vb = 10.728·Qts^2.4186·Vas       Fb = 0.3802·Qts^-1.0657·Fs
Zbinden(M4): Vb = (2.52·Qts-0.35)·Vas         Fb = 0.32·√(1/Qts²+3.38)·Fs
"max-flat":  Vb = 20·Vas·Qts^3.3              Fb = Fs·(Vas/Vb)^0.31

Scimpy adds Chebyshev (C4) and QB3 (find_ported_params_c4, find_ported_params_qb3). Analysis: these trace to public papers (Bullock, Keele/Hoge, Zbinden) — the formulas are facts, not copyrightable; only the GPL code is.

Recommendation: implement Bessel / Bullock / Keele-Hoge / Legendre / Chebyshev / QB3 / SBB4 presets in the engine, reimplemented from the published formulas (cite the papers; do not copy QSpeakers/Scimpy GPL source). Cross-check each preset's output against these two tools' numbers as a free oracle.

F3 — Box-loss (Ql) belongs in the transfer function GAP

QSpeakers' ported response() threads a loss quality factor through every coefficient (getLossQualityFactor()); SpeakerBoxLite defaults Ql=7; lautsprechershop bakes a leakage term into qr (F1). OpenISD's loss model is partial (Qa is backlog).

Recommendation: make Ql (and Qa/Qp) first-class inputs and route them into the vented/bandpass transfer function exactly as QSpeakers does (formula visible in system.cpp). Also lets OpenISD match other tools' lossy output when cross-checking (set Ql high = OpenISD's current lossless default).

F4 — OpenISD's sealed/bandpass math is corroborated by open source CONFIDENCE

QSpeakers' sealed response is 10·log10(fr²/((fr−1)²+fr/Qtc²)) with Qtc=√(1+Vas/Vb)·Qts, fb=√(1+Vas/Vb)·Fs — identical to OpenISD's sealed model. Its 4th-order ported & bandpass transfer functions are fully written out and inspectable.

Recommendation: add a unit test that evaluates OpenISD's sealed/ported/bandpass response at a few frequencies and asserts it against QSpeakers' formula (reimplemented in the test) — an open, inspectable oracle for the curve shape, complementing micka's scalar checks. Far more trustworthy than scraping SBL.

F5 — The open-source field is thin and mostly abandoned POSITIONING

Of the OSS box tools: QSpeakers is active (last commit 2026-03, 54★) but Linux/Qt desktop; Scimpy is abandoned (2017, 72★); GSpeakers is legacy. None has a test suite (Scimpy's speakertest.py is a soundcard measurement engine, not unit tests). The dominant tools are all closed Windows freeware.

Recommendation: OpenISD is the only modern, web-native, open, tested tool in the space — lean into that as the differentiator. Track QSpeakers (active GPL formulas) as the reference to cross-check against; don't depend on Scimpy/GSpeakers (dead).

F6 — There is a trust vacuum around free tools OPPORTUNITY

Users report SpeakerBoxLite driver params "WAY off" and low confidence in free web apps' accuracy; even respected tools disagree on bandpass.

Recommendation: make verifiability OpenISD's brand: publish the formulas + literature citations in-app, keep the golden/oracle tests visible, and show which alignment each result uses. That directly answers the community's #1 complaint (unvetted black-box accuracy).
